React crea una representación virtual del DOM\n2. Cuando hay cambios, React actualiza el Virtual DOM\n3. Compara (diff) el Virtual DOM con el DOM real\n4. Solo actualiza los elementos que cambiaron\n\nVentajas:\n- Detecta cambios eficientemente\n- Solo actualiza elementos necesarios\n- Mayor velocidad y rendimiento\n- Menos consumo de recursos\n- Mejor experiencia de usuario\n\n---\n\n### 4. ¿Qué es JSX?\n\nJSX permite escribir código HTML dentro de archivos JavaScript, facilitando la creación de componentes.\n\nVentajas:\n- Sintaxis más clara y legible\n- Similar a HTML tradicional\n- Poder de JavaScript combinado con HTML\n- Mejor mantenibilidad\n\nEjemplo en nuestro proyecto:\n```jsx\nreturn (\n  <div className=\"tarjeta\">\n    <h1>{datosPersonales.nombre}</h1>\n    <img src={datosPersonales.foto} alt=\"Perfil\" />\n    <button onClick={handleContacto}>Contacto</button>\n  </div>\n);\n```\n\nReglas JSX:\n- Un solo elemento padre\n- Usar className en lugar de class\n- Las etiquetas deben cerrarse: <img />\n- Usar expresiones JavaScript con {}\n\n---\n\n### 5. ¿Qué es un componente?\n\nUn componente es una pieza reutilizable de la interfaz que encapsula HTML, CSS y lógica JavaScript.\n\nEn este proyecto:\n- TarjetaPersonal.jsx es un componente que muestra la información personal\n- Puede reutilizarse para mostrar diferentes personas\n- Contiene su propia lógica y estilos\n\nVentajas de los componentes:\n- Reutilización\n- Modularidad\n- Encapsulación\n- Código más limpio\n- Fácil mantenimiento\n\nEstructura de un componente:\n```jsx\nfunction NombreComponente() {\n  // Lógica y datos\n  \n  return (\n    // JSX\n  );\n}\n\nexport default NombreComponente;\n```\n\n---\n\n##
